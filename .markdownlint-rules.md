# Markdown Lint Configuration

## Current Rules (.markdownlint.json)

```json
{
  "default": true, // Enable all rules by default
  "MD013": false, // Line length - Allow long lines (URLs, tables)
  "MD024": {
    "siblings_only": true // Duplicate headings - OK in different sections
  },
  "MD029": false, // Ordered list prefix - Allow semantic numbering (1,2,3...7,8,9)
  "MD031": false, // Blanks around fences - Allow tight spacing in lists
  "MD034": false, // Bare URLs - Allow emails and inline URLs
  "MD036": false, // Emphasis as heading - Allow bold for subsections
  "MD040": false, // Fenced code language - Allow blocks without language
  "MD041": false // First line heading - Allow intro text before title
}
```

## Quick Reference

| Rule  | Name                   | Status   | Reason                                   |
| ----- | ---------------------- | -------- | ---------------------------------------- |
| MD013 | Line length            | Disabled | Long URLs and code examples              |
| MD024 | No duplicate headings  | Modified | Allow duplicates in different sections   |
| MD029 | Ordered list prefix    | Disabled | Allow semantic multi-step numbering      |
| MD031 | Blanks around fences   | Disabled | Compact formatting in lists              |
| MD034 | No bare URLs           | Disabled | Email addresses look better without `<>` |
| MD036 | No emphasis as heading | Disabled | Bold text for visual hierarchy           |
| MD040 | Fenced code language   | Disabled | Plain text/output doesn't need language  |
| MD041 | First line heading     | Disabled | Allow intro paragraphs before headings   |

## Examples

### MD013 - Line Length ✅ Allowed

```markdown
Database URL: postgresql://username:password@localhost:5432/my_really_long_database_name_that_exceeds_80_characters
```

### MD024 - Duplicate Headings ✅ Allowed

```markdown
## Module A

### Overview ← First "Overview"

## Module B

### Overview ← Duplicate OK (different parent)
```

### MD029 - Semantic Numbering ✅ Allowed

````markdown
1. Update schema
2. Create migration

   ```bash
   npm run migrate:dev
   ```
````

3. Verify changes ← Continues from 3, not restarting at 1
4. Deploy

````

### MD031 - Tight Spacing ✅ Allowed
```markdown
1. **Run command:**
   ```bash
   npm test
````

2. **Check output:**

````

### MD034 - Bare URLs ✅ Allowed
```markdown
Contact: admin@example.com
Database: postgresql://localhost:5432
````

### MD036 - Bold Emphasis ✅ Allowed

```markdown
### Solutions

**Option 1: Docker** ← Bold for emphasis, not a heading
Use Docker for...

**Option 2: Local**
Install locally...
```

### MD040 - No Language ✅ Allowed

````markdown
```
Output from command:
  File processed successfully ✓
```
````

### MD041 - Intro Text ✅ Allowed

```markdown
This guide explains the API endpoints.

Last updated: Dec 21, 2025

# API Documentation ← Heading not on line 1
```
