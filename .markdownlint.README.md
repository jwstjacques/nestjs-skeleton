# .markdownlint.json - Configuration Reference

This file disables certain markdown lint rules that don't fit our documentation style.

## Disabled Rules

```
MD013 - Line length           → Allow long URLs and code examples
MD024 - Duplicate headings     → Allow same heading in different sections
MD029 - Ordered list prefix    → Allow semantic numbering (1,2,3...7,8,9)
MD031 - Blanks around fences   → Allow compact code blocks in lists
MD034 - Bare URLs              → Allow email addresses without brackets
MD036 - Emphasis as heading    → Allow bold text for visual hierarchy
MD040 - Fenced code language   → Allow plain text code blocks
MD041 - First line heading     → Allow intro text before title
```

## Full Documentation

See `.markdownlint-rules.md` for detailed explanations and examples of each rule.

## Quick Edit

To re-enable a rule, change its value from `false` to `true` in `.markdownlint.json`.

To modify MD024 (duplicate headings), change `"siblings_only": true` to `false`.
